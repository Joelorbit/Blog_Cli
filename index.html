blog-cli

Publish Markdown Files as Blog Posts from Your Terminal
Version 1.0.0 — © 2026 Joelorbit — MIT License

● Markdown In, HTML Out

Write in plain .md files. blog-cli converts to HTML using GitHub Flavored Markdown. Supports tables, task lists, strikethrough.

● Git-Powered

Every publish commits your post and pushes to GitHub. No server needed. Your website fetches JSON straight from the repo.

● Filename = Title

Name your file, get a title. my-first-post.md becomes "My First Post". Re-publish to update. Same slug, new content.

● Per-User Config

Each user points to their own repo. blog.json stores YOUR remote URL. Your posts never touch anyone else's repo.

■ How It Works

1
Clone this repo and run npm install && npm link
2
Go to your project: cd /path/to/your/portfolio
3
Initialize with your repo: blog init --repo https://github.com/you/your-blog.git
4
Write a markdown file and publish: blog publish ~/notes/my-post.md
5
Fetch from your website using GitHub raw URLs

💻 Quick Install

# Clone and install git clone https://github.com/Joelorbit/Blog_Cli.git cd Blog_Cli npm install npm link # Set up your blog blog init --repo https://github.com/you/your-blog.git # Publish a post blog publish ~/notes/my-first-post.md # List posts blog list

blog.json       ← { postsDir, remote, branch }
